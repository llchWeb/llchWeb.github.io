阿里云服务器快速搭建hexo博客

建议大家在Xshell6软件中操作，当然如果是在您自己的浏览器控制台远程连接操作也可以
先查看是否已经安装有git和node.js，如下所示。
如果有，则忽略一、二步。
如果没有则继续安装

[root@izuf67ehbr4ubh35o7bdctz local]# node -v 检查是否安装成功node.js
v9.11.0
[root@izuf67ehbr4ubh35o7bdctz local]# npm -v 检查是否安装成功npm
5.6.0

第一步，安装git

$ sudo yum install git-core

/usr/libexec/git-core
/etc/bash_completion.d/git

第二步，安装node.js

（通过网站https://nodejs.org/dist/v9.11.0/ 下载node-v5.10.1-linux-x64.tar.gz的包到widow操作系统中，然后通过rz的命令将文件传输到服务器上）

如果您没有rz命令，可以百度查看Linux操作系统如何安装lrzsz即可。网上有许多教程，我这就不啰嗦了

最好不要安装源码文件（Source Code）https://nodejs.org/en/download/
而是安装编译后的文件 https://nodejs.org/dist/v9.11.0/
解压后，在bin文件夹中已经存在node以及npm，如果你进入到对应文件的中执行命令行一点问题都没有，不过不是全局的，所以通过建立软链接的方式将这个设置为全局就好了

cd到你的安装包目录 我的是cd /home/program_file/download/
tar -zxvf node-v5.10.1-linux-x64.tar.gz -C /usr/local/
cd /usr/local/
mv node-v5.10.1-linux-x64/ nodejs 这个相当于重命名
建立软链接，相当于是桌面快捷方式
ln -s /usr/local/nodejs/bin/node /usr/local/bin
ln -s /usr/local/nodejs/bin/npm /usr/local/bin

第三步，安装hexo

$ npm install -g hexo-cli      ##博客引擎的命令行工具

将hexo添加到全局
ln -s /usr/local/nodejs/lib/node_modules/hexo-cli/bin/hexo /usr/local/bin/hexo

第四步，建立站点

$ cd /usr/local
hexo init small-blog            ##small-blog就是一个网站floder,init命令，新建一个网站。

如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站
cd small-blog
npm install 安装需要的依赖包

第五步，配置

$ cd /usr/local/small-blog       ##cd 你的网站文件夹
$ vim _config.yml   ##编辑配置文件

缩进
YAML依靠缩进来确定元素间的从属关系。因此，请确保每个deployer的缩进长度相同，并且使用空格缩进。也就是每个“:”后面加一个空格

title: 超浪贼六的博客
subtitle: 刘浪超的技术博客，当然这也是用于和大家学习交流的平台，望各位多多关照！
description: 描述部分，便于搜索引擎查询。例如我的是“技术博客。并发编程。synchronized锁。Java视频教程”
author: 阿超
然后保存退出，按esc，然后:wq回车即可

第六步：命令

$ hexo new "我的第一篇博客"

显示提示信息 INFO Created: /usr/local/small-blog/source/_posts/我的第一篇博客.md
$ hexo new [layout] <br>新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。<br>$ hexo new ”post title with whitespace“<br>输入内容，保存退出

hexo generate 该命令可以简写为 hexo g
生成静态文件。(在网站目录下，有public目录生成)
选项 描述
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动
相当于是在source里去写一个文件，通过hexo g自动生成到public目录中

第七步：启动测试环境

$ hexo server

启动服务器。默认情况下，访问网址为： http://localhost:4000/
阿里云服务器则为http://你的公网ip:4000/
选项 描述
-p, –port 重设端口
-s, –static 只使用静态文件
-l, –log 启动日记记录，使用覆盖记录格式

如果是服务器，我的是阿里云服务器，需要配置安全组打开4000端口,或者是在防火墙开启4000的端口，这样就可以在服务器本地访问,如果是linux虚拟机，需要测试内外能否正常通信,输入ip:4000/ 即可访问hexo的初始页面

以上的内容是按照 https://www.bilibili.com/video/av55851824/?p=1 视频教程编写的。
个人非常感谢匠码笔记

以上的操作已经完成了在阿里云服务器上的博客平台搭建。
以后只要是想要发布博客文章。只需要hexo new “你的博客标题” 然后输入内容即可
如果想要修改内容，cd 你的网站目录/source/_posts 例如我的就是/usr/local/small-blog/source/_posts
然后vim 你的博客标题.md即可
如果想要查看如何发布到GitHub上，则请阅读其他文章
